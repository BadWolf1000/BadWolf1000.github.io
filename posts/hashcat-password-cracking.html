Mastering Hashcat: The Complete Guide to Password Cracking

Hashcat is the world's fastest and most advanced password recovery utility, supporting hundreds of hash types and offering incredible flexibility for security professionals. Whether you're conducting penetration tests, recovering forgotten passwords, or auditing password policies, Hashcat is an essential tool in your arsenal.

What is Hashcat?

Hashcat is a powerful open-source password cracker that leverages the massive parallel processing power of GPUs to crack password hashes at incredible speeds. It supports over 300 hash types including MD5, SHA-1, SHA-256, NTLM, bcrypt, and many more specialized formats like Kerberos tickets and cryptocurrency wallets.

Installation

On most Linux distributions, you can install Hashcat from the package manager:

# Debian/Ubuntu
sudo apt install hashcat

# Arch Linux
sudo pacman -S hashcat

# Or download the latest from hashcat.net
wget https://hashcat.net/files/hashcat-6.2.6.7z
7z x hashcat-6.2.6.7z

For optimal performance, ensure you have the latest GPU drivers installed (NVIDIA CUDA or AMD ROCm).

Understanding Hash Types

Hashcat uses numeric mode identifiers for different hash types. Here are some common ones:

  • -m 0 - MD5
  • -m 100 - SHA-1
  • -m 1400 - SHA-256
  • -m 1000 - NTLM (Windows passwords)
  • -m 3200 - bcrypt
  • -m 1800 - sha512crypt (Linux /etc/shadow)
  • -m 13100 - Kerberos 5 TGS-REP (Kerberoasting)
  • -m 18200 - Kerberos 5 AS-REP (AS-REP Roasting)
  • -m 5600 - NetNTLMv2
  • -m 22000 - WPA-PBKDF2-PMKID+EAPOL

You can view all supported hash types with:

hashcat --help | grep -i "hash modes"
hashcat --example-hashes

Attack Modes

Hashcat supports several attack modes, each suited for different scenarios:

Dictionary Attack (Mode 0)

The most common attack type. Uses a wordlist to try passwords:

hashcat -m 0 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt

Combination Attack (Mode 1)

Combines words from two dictionaries:

hashcat -m 0 -a 1 hashes.txt wordlist1.txt wordlist2.txt

Brute-Force Attack (Mode 3)

Tries all possible combinations using a mask:

# Try all 8-character lowercase passwords
hashcat -m 0 -a 3 hashes.txt ?l?l?l?l?l?l?l?l

Rule-Based Attack

Applies transformation rules to wordlist entries:

hashcat -m 0 -a 0 hashes.txt rockyou.txt -r /usr/share/hashcat/rules/best64.rule

Mask Attack Deep Dive

Masks are powerful for targeted brute-force attacks. Character sets include:

  • ?l - lowercase letters (a-z)
  • ?u - uppercase letters (A-Z)
  • ?d - digits (0-9)
  • ?s - special characters
  • ?a - all printable ASCII
  • ?b - all bytes (0x00-0xff)

Examples of effective masks:

# Password + 4 digits (common pattern)
hashcat -m 1000 hashes.txt -a 3 ?u?l?l?l?l?l?l?l?d?d?d?d

# Company name + year + special char
hashcat -m 1000 hashes.txt -a 3 Company2024?s

# Custom charset: only specific chars
hashcat -m 0 hashes.txt -a 3 -1 ?l?d ?1?1?1?1?1?1?1?1

Using Rules Effectively

Rules transform wordlist entries to match common password patterns. Some built-in rule files:

  • best64.rule - Top 64 most effective rules
  • rockyou-30000.rule - Generated from RockYou patterns
  • dive.rule - Comprehensive ruleset
  • OneRuleToRuleThemAll.rule - Community favorite

Common rule functions:

# Rule syntax examples
:       # Do nothing (passthrough)
l       # Lowercase all
u       # Uppercase all
c       # Capitalize first, lowercase rest
$1      # Append '1'
^!      # Prepend '!'
sa@     # Substitute 'a' with '@'
ss$     # Substitute 's' with '$'

Create a custom rule file:

# custom.rule
:
$1
$123
$!
c
c$1
sa@se3si1so0

Real-World Examples

Cracking NTLM Hashes from SAM

After extracting NTLM hashes from a Windows system:

# Extract with secretsdump.py
secretsdump.py -sam SAM -system SYSTEM LOCAL

# Crack the hashes
hashcat -m 1000 ntlm_hashes.txt rockyou.txt -r best64.rule

Kerberoasting Attack

Crack TGS tickets obtained from Kerberoasting:

# Get tickets with GetUserSPNs.py
GetUserSPNs.py domain/user:password -dc-ip 10.10.10.1 -request

# Crack the tickets
hashcat -m 13100 kerberos_hashes.txt rockyou.txt -r best64.rule

AS-REP Roasting

For accounts with "Do not require Kerberos preauthentication":

# Get hashes
GetNPUsers.py domain/ -usersfile users.txt -dc-ip 10.10.10.1

# Crack
hashcat -m 18200 asrep_hashes.txt rockyou.txt

Cracking NetNTLMv2 from Responder

After capturing hashes with Responder:

hashcat -m 5600 responder_hashes.txt rockyou.txt -r dive.rule

Wi-Fi WPA2 Handshakes

Convert capture to hashcat format and crack:

# Convert with hcxpcapngtool
hcxpcapngtool -o hash.hc22000 capture.pcapng

# Crack
hashcat -m 22000 hash.hc22000 rockyou.txt

Performance Optimization

Get the most out of your hardware:

# Show device info
hashcat -I

# Use specific GPU
hashcat -d 1 -m 0 hashes.txt wordlist.txt

# Optimize workload (1-4, higher = faster but less responsive)
hashcat -w 3 -m 0 hashes.txt wordlist.txt

# Use optimized kernels (faster but limits password length)
hashcat -O -m 0 hashes.txt wordlist.txt

Benchmarking

Test your system's cracking speed:

# Benchmark all hash types
hashcat -b

# Benchmark specific hash type
hashcat -b -m 1000

Session Management

Hashcat can pause and resume cracking sessions:

# Start with session name
hashcat -m 0 hashes.txt wordlist.txt --session=crack1

# Restore a session
hashcat --session=crack1 --restore

# Show session status
hashcat --session=crack1 --status

During a running session, press:

  • s - Show status
  • p - Pause
  • r - Resume
  • c - Checkpoint (save and quit)
  • q - Quit

Output Options

Control how cracked passwords are displayed and saved:

# Output to file
hashcat -m 0 hashes.txt wordlist.txt -o cracked.txt

# Output format (default: hash:password)
hashcat -m 0 hashes.txt wordlist.txt -o cracked.txt --outfile-format=2

# Show cracked passwords
hashcat -m 0 hashes.txt --show

# Show only uncracked
hashcat -m 0 hashes.txt --left

Potfile Management

Hashcat stores cracked hashes in a potfile to avoid re-cracking:

# Default location: ~/.local/share/hashcat/hashcat.potfile

# Disable potfile
hashcat -m 0 hashes.txt wordlist.txt --potfile-disable

# Use custom potfile
hashcat -m 0 hashes.txt wordlist.txt --potfile-path=custom.pot

Advanced Techniques

Prince Attack

Generate password candidates by combining words:

# Install princeprocessor
./pp64.bin wordlist.txt | hashcat -m 0 hashes.txt

Hybrid Attacks

Combine dictionary with mask:

# Wordlist + mask (mode 6)
hashcat -m 0 -a 6 hashes.txt wordlist.txt ?d?d?d?d

# Mask + wordlist (mode 7)
hashcat -m 0 -a 7 hashes.txt ?d?d?d?d wordlist.txt

Loopback Attack

Use cracked passwords as input for further cracking:

hashcat -m 0 hashes.txt wordlist.txt --loopback

Building Effective Wordlists

Good wordlists are crucial for success:

  • rockyou.txt - Classic, 14 million passwords
  • SecLists - Comprehensive collection
  • OSINT-gathered words from target (company names, locations, products)
  • Previous breach data (legally obtained)
# Combine and dedupe wordlists
cat wordlist1.txt wordlist2.txt | sort -u > combined.txt

# Generate targeted wordlist with CeWL
cewl -d 3 -m 5 https://target.com -w custom_wordlist.txt
Always ensure you have proper authorization before attempting to crack any passwords. Unauthorized password cracking is illegal. Use these techniques only in legitimate penetration testing engagements, CTF competitions, or to recover your own passwords.

Troubleshooting Common Issues

  • Token length exception - Hash format is wrong or contains extra characters
  • No hashes loaded - Check hash format matches the mode (-m)
  • Separator unmatched - For hashes with usernames, use --username flag
  • GPU temp throttling - Improve cooling or use --gpu-temp-abort

Conclusion

Hashcat is an incredibly powerful tool that rewards practice and experimentation. Start with basic dictionary attacks, then progressively incorporate rules, masks, and hybrid attacks. Understanding password patterns and building targeted wordlists will dramatically improve your success rate.
