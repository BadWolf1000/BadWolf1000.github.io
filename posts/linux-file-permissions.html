Understanding Linux File Permissions

File permissions are a fundamental concept in Linux security. They control who can read, write, and execute files on your system. Understanding them is essential for any Linux user or administrator.

The Permission Model

Linux uses a permission model based on three types of access:

  • Read (r) - View file contents or list directory contents
  • Write (w) - Modify file contents or create/delete files in a directory
  • Execute (x) - Run a file as a program or access a directory

Permission Groups

These permissions are assigned to three groups:

  • Owner (u) - The user who owns the file
  • Group (g) - The group that owns the file
  • Others (o) - Everyone else

Reading Permissions

Use ls -l to view permissions:

$ ls -l script.sh
-rwxr-xr-- 1 ryan users 1024 Jan 25 10:00 script.sh

Breaking down -rwxr-xr--:

  • - - File type (- for regular file, d for directory)
  • rwx - Owner permissions (read, write, execute)
  • r-x - Group permissions (read, execute)
  • r-- - Others permissions (read only)

Numeric Permissions

Permissions can also be expressed as numbers:

  • Read = 4
  • Write = 2
  • Execute = 1

Add them up for each group. So rwxr-xr-- becomes 754.

Changing Permissions

Use chmod to change permissions:

# Using symbolic notation
chmod u+x script.sh        # Add execute for owner
chmod g-w file.txt         # Remove write for group
chmod o=r file.txt         # Set others to read only

# Using numeric notation
chmod 755 script.sh        # rwxr-xr-x
chmod 644 file.txt         # rw-r--r--

Changing Ownership

Use chown to change the owner and group:

chown ryan file.txt           # Change owner
chown ryan:users file.txt     # Change owner and group
chown :users file.txt         # Change only group

Special Permissions

Linux also has special permissions:

  • SUID (4) - Run as file owner
  • SGID (2) - Run as group owner
  • Sticky Bit (1) - Only owner can delete in directory
Be careful with SUID/SGID on executables - they can be security risks if not properly managed.

Best Practices

  • Apply the principle of least privilege
  • Regularly audit file permissions
  • Be cautious with 777 permissions
  • Use groups to manage access efficiently

Conclusion

Mastering file permissions is crucial for Linux security. Take time to understand them, and always think carefully before changing permissions on sensitive files.
